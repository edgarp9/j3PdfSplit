# PDF 분할 및 선택 페이지 내보내기 요구사항 정본

- 정본 여부: 정본
- 범위: Tkinter 기반 PDF 분할 및 선택 페이지 내보내기 애플리케이션의 기능, UI, 저장 규칙, 수용 기준

## 목적

사용자가 PDF를 열고 전체 페이지를 썸네일로 확인한 뒤, 상단 모드에 따라 다음 작업을 수행한다.

1. 순차 분할 모드: 특정 페이지를 순서대로 선택하면서 PDF를 겹치지 않는 여러 개의 파일로 분할 저장한다.
2. 선택 페이지 내보내기 모드: 여러 페이지를 선택하고, 선택한 페이지만 한 PDF로 내보낸다.

## 모드 규칙

1. 상단에 `순차 분할`과 `선택 페이지 내보내기` 두 가지 모드가 있어야 하며, 기본값은 `선택 페이지 내보내기`이다.
2. `순차 분할` 모드는 기존 순차 분할 규칙을 그대로 따른다.
3. `선택 페이지 내보내기` 모드에서는 썸네일 클릭이 즉시 저장을 실행하지 않고 페이지 선택/해제를 토글해야 한다.
4. `선택 페이지 내보내기` 모드에서는 `내보내기` 버튼을 눌렀을 때만 실제 파일을 생성한다.
5. 선택 페이지는 원본 문서의 페이지 순서대로 내보낸다.
6. 선택 페이지 내보내기는 순차 분할 진행 상태를 변경하지 않는다.

## 핵심 규칙

1. 페이지 번호는 모두 `0`부터 시작하는 인덱스로 처리한다.
2. 첫 저장 구간은 항상 `0`페이지부터 시작한다.
3. 이후 저장 구간은 직전 저장 구간의 마지막 페이지 다음 페이지부터 시작한다.
4. 사용자는 이미 저장한 마지막 페이지보다 이전 페이지를 다시 선택할 수 없다.
5. 마지막 페이지까지 저장되면 더 이상 추가 분할을 허용하지 않는다.

## 예시

- `3`페이지 선택 -> `0~3` 저장
- `5`페이지 선택 -> `4~5` 저장
- `8`페이지 선택 -> `6~8` 저장

## UI 요구사항

1. `PDF 열기` 버튼이 있어야 한다.
2. 상단에 모드 선택 UI가 있어야 한다.
3. 썸네일은 스크롤 가능한 영역에 표시한다.
4. 상단에 `10%`부터 `200%`까지 조절 가능한 썸네일 크기 선택 UI가 있어야 하며, 초기값은 `70%`여야 한다.
5. 각 썸네일 아래에 0부터 시작하는 페이지 번호를 표시한다.
6. 다음 분할 시작 페이지를 화면에 표시한다.
7. 이미 저장 완료된 페이지와 미저장 페이지를 시각적으로 구분한다.
8. 저장 폴더는 원본 폴더 기본값을 사용하되, 사용자가 다른 폴더로 변경할 수 있어야 한다.
9. 썸네일은 페이지 내용을 식별하기 쉬운 크기로 표시한다.
10. 썸네일 크기를 바꾸면 현재 문서의 썸네일이 새 크기로 다시 렌더링되어야 한다.
11. 썸네일 로딩 중에도 UI 스크롤과 버튼 반응이 크게 저하되지 않아야 한다.
12. 대용량 문서는 현재 화면에 보이는 페이지와 인접 페이지의 썸네일부터 우선 렌더링되어야 한다.
13. `순차 분할` 모드에서는 썸네일 선택 시 저장 범위, 파일명, 저장 폴더를 먼저 보여 주고 실제 저장 여부를 확인받아야 한다.
14. `선택 페이지 내보내기` 모드에서는 선택된 페이지 수와 내보내기 가능 여부를 화면에서 확인할 수 있어야 한다.
15. `선택 페이지 내보내기` 모드에서는 `내보내기` 버튼이 있어야 한다.
16. UI 언어는 메뉴에서 영어와 한국어 중 선택할 수 있어야 하며, 기본값은 영어여야 한다.
17. 언어를 변경하면 버튼, 라벨, 상태 메시지, 확인 대화상자, 썸네일 상태 문구가 선택한 언어로 표시되어야 한다.
18. 메인 윈도우 타이틀바 제목은 `j3PdfSplit`이어야 한다.
19. 메뉴에서 About 창을 열 수 있어야 한다.
20. About 창에는 버전 정보를 표시하고, 하단에 `https://github.com/edgarp9` 링크를 표시해야 한다.
21. About 창 하단 링크를 누르면 기본 브라우저로 해당 링크를 열어야 한다.
22. About 창은 메인 윈도우 기준 중앙에 모달로 표시되어야 한다.
23. 메뉴에서 `PDF 열기`, `저장 폴더 선택`, `내보내기` 명령을 실행할 수 있어야 한다.
24. 상단 작업 버튼은 `PDF 열기`, `저장 폴더 선택`, `내보내기` 순서로 표시되어야 한다.
25. 메인 화면 상단 헤더에는 앱 제목 라벨을 표시하지 않는다.
26. 썸네일 영역 위에는 페이지 상태 범례와 모드 사용 안내 라벨을 표시하지 않는다.
27. 상단 헤더는 실행 작업과 설정을 분리한다. 첫 번째 행에는 작업 버튼을 배치하고, 두 번째 행에는 모드 선택과 썸네일 크기 조절을 배치한다.

## 저장 규칙

1. 파일명은 원본 파일명을 기준으로 순서가 드러나야 한다.
2. 순차 분할 모드의 기본 형식은 `원본파일명_1장.pdf`, `원본파일명_2장.pdf`처럼 저장한다.
3. 선택 페이지 내보내기 모드의 기본 파일명은 `원본파일명_선택페이지.pdf`이다.
4. 같은 이름의 파일이 이미 있으면 덮어쓰지 않고 고유 이름으로 저장한다.

## 예외 처리

1. PDF를 열기 전에는 페이지 선택 동작이 실행되지 않아야 한다.
2. 이미 저장한 페이지보다 이전 페이지를 선택하면 경고 메시지를 보여 주고 저장하지 않는다.
3. 잘못된 PDF, 읽기 실패, 저장 실패는 사용자 메시지와 내부 로그로 구분해 처리한다.
4. 마지막 페이지까지 저장한 후에는 완료 메시지를 보여 준다.
5. 선택 페이지 내보내기 모드에서 아무 페이지도 선택하지 않고 `내보내기`를 누르면 경고 메시지를 보여 주고 저장하지 않는다.

## 수용 기준

- PDF를 열면 전체 페이지 수만큼 썸네일 카드가 생성된다.
- 상단 모드 선택 UI의 기본값은 `선택 페이지 내보내기`이다.
- 상단 썸네일 크기 선택 UI는 초기값 `70%`로 표시되고, `10%`부터 `200%` 사이 값을 적용할 수 있다.
- 썸네일 크기를 바꾸면 기존 문서의 썸네일 카드가 새 크기로 다시 표시된다.
- `순차 분할` 모드에서 썸네일 클릭 시 현재 시작 페이지부터 선택 페이지까지의 PDF가 정확히 한 파일로 저장된다.
- 저장이 성공하면 다음 시작 페이지가 즉시 갱신된다.
- 저장 완료 카드와 미완료 카드의 시각 상태가 갱신된다.
- 이미 완료한 구간을 다시 선택하면 파일이 생성되지 않는다.
- 썸네일 로딩 중에도 창이 멈추지 않고 계속 반응한다.
- 대용량 문서를 열면 현재 화면 근처의 썸네일이 먼저 표시되고, 스크롤에 따라 다음 썸네일이 이어서 로딩된다.
- `순차 분할` 모드에서는 썸네일 선택 후 사용자가 확인을 눌렀을 때만 파일이 생성된다.
- `선택 페이지 내보내기` 모드에서는 썸네일 클릭으로 선택/해제가 가능하다.
- `선택 페이지 내보내기` 모드에서 `내보내기`를 누르면 선택한 페이지들만 한 PDF로 저장된다.
- 선택 페이지 내보내기 후에도 순차 분할의 다음 시작 페이지는 변경되지 않는다.
- 앱을 처음 실행하면 영어 UI로 표시된다.
- 사용자가 메뉴에서 언어를 한국어 또는 영어로 바꾸면 현재 화면의 UI 문구가 즉시 갱신된다.
- About 메뉴를 누르면 버전 정보와 `https://github.com/edgarp9` 링크가 있는 모달 창이 열린다.
- File 메뉴에서 `PDF 열기`, `저장 폴더 선택`, `내보내기`를 실행할 수 있다.
- 상단 작업 버튼은 `PDF 열기`, `저장 폴더 선택`, `내보내기` 순서로 표시된다.
